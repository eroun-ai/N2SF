N2SF란 무엇인가?

국가 망 보안체계(N2SF: National Network Security Framework)는 기존 획일적 망 분리 정책을 넘어, 업무정보의 중요도에 따라 기밀(C), 민감(S), 공개(O) 등급으로 분류하고, 등급별로 차등적인 보안통제를 적용하는 보안 프레임워크입니다.

핵심 변화: 망 분리(물리적 분리) 중심 → 업무정보 중요도 기반 차등 보안통제 적용

추진 목표 5가지

1. 망 분리 중심에서 N2SF 기반 보안정책으로 패러다임 전환
2. AI·클라우드 등 신기술 융합으로 스마트 업무환경 실현
3. 산학연관 협력으로 기관 자율보안 접목
4. 기관별 자율 추진 지원, 관련 제도 점진적 개선
5. 공공정보의 보안성·활용성 동시 제고, 디지털 경제 활성화

적용 대상 및 시기

적용 대상: 국가·공공기관의 모든 정보화사업 (신규 구축 및 기존 운영 중 정보서비스의 N2SF 적용 포함)
가이드라인 버전: 1.0 (2025년 9월 발간)
참고 문서: 부록1(보안통제 항목 해설서), 부록2(정보서비스 모델 해설서 1~11), AI보안 가이드북

C/S/O 등급분류 기준

등급분류기준정보공개법 근거
C 기밀비밀, 안보·국방·외교·수사 등 기밀정보, 국민 생명·안전 직결 정보제9조 제1호~제4호
S 민감감사·감독·검사·인사관리 관련 정보, 개인정보, 경영·영업 비밀 등제9조 제5호~제8호 + 기타(로그, 임시백업)
O 공개C·S 외 모든 정보, 비공개 필요성 소멸 정보그 외

등급분류 절차

1단계: 업무정보 분류 → 2단계: 정보시스템 분류 (업무정보 등급 기반) → 3단계: 위치 분류 (물리적·논리적 영역)

주의: 하나의 정보시스템에 서로 다른 등급의 업무정보가 포함될 경우, 가장 높은 등급을 해당 시스템의 등급으로 분류합니다. 가급적 동일 등급별로 시스템을 분리 운영하는 것을 권고합니다.

위치-주체-객체 모델링

정보서비스 환경을 위치(Domain)-주체(Subject)-객체(Object)로 단순화하여 위협을 식별합니다.

주체(Subject): 업무정보를 활용하는 정보시스템 (단말 등)
객체(Object): 주체가 접속/이용하는 대상 정보시스템 (서버 등)
위치(Domain): 주체가 위치한 물리적·논리적 영역

각 위치·주체·객체에 C/S/O 등급을 부여한 후, 등급이 서로 다를 경우 보안대책이 필요합니다.

정보 생산·저장 원칙

정보시스템은 자신의 보안등급과 동일하거나 낮은 등급의 업무정보를 생산·저장할 수 있습니다.

위협 예시: O등급 생성형 AI 서비스에서 S등급 업무정보가 생산·저장되는 경우 → 보안대책 필요

정보 이동 원칙

업무정보는 자신의 보안등급과 동일하거나 높은 등급의 정보시스템으로 이동할 수 있습니다.

위협 예시: S등급 단말에서 O등급 서비스로 S등급 정보를 업로드하는 경우 → 보안대책 필요
